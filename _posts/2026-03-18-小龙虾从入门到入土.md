## Openclaw安装与部署
由于小龙虾是顶流，网上部署教程特别多。 以下按照[部署教程](https://docs.openclaw.ai/zh-CN/start/getting-started#macos%2Flinux)来做。

1. 前期准备
安装OpenClaw需要node.js, 官方推荐使用 Node 24（Node 22 LTS，目前为 22.16+，仍因兼容性而受支持）
> 我是macos，本地node.js的版本是22.16.0

2. 开始安装
- 终端输入命令
```
curl -fsSL https://openclaw.ai/install.sh | bash 
```
> -
问题：我本地的node.js总有问题，从使用这个chirpy主题的博客时就开始了。此次安装环节第三步又报错如下：
```
npm install failed for openclaw@latest
Command: env SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm --loglevel error --silent --no-fund --no-audit install -g openclaw@latest
```
千问老师指导，输入命令`cat ~/.npm/_logs/$(ls -t ~/.npm/_logs/ | head -1) | tail -100`查看了npm全局报错日志，显示如下（截取）：
```
19 error code UNABLE_TO_GET_ISSUER_CERT_LOCALLY
20 error errno UNABLE_TO_GET_ISSUER_CERT_LOCALLY
21 error request to https://registry.npmmirror.com/openclaw failed, reason: unable to get local issuer certificate
```
搜索原因是 npm 在验证其尝试连接的注册表的 SSL 证书时遇到了困难。此问题通常在使用自签名证书时，或 npm 的 SSL 设置存在配置错误时出现。

这里采取临时处理措施，注意只在信任的网络环境这样做。
```
npm config set strict-ssl false

# 安装完恢复
npm config set strict-ssl true
```

3. 安装完成
部署教程里显示的新手引导命令：
```
openclaw onboard --install-daemon
```
这个命令的输出其实在上述安装过程中就会自动执行了。输出会展示安全协议，配置Gateway、大模型api，引导安装官方skills,安装hooks等，为了后续调整，我能选`skip for now`的就都选了，skill安装了summarize，hooks全部安装。 

此时基本部署好了，接着去找合适的channel和provider

## 飞书与Minimax

### 配置Minimax 
参考教程[openclaw配置minimax M2.7](https://platform.minimaxi.com/docs/token-plan/openclaw#%E9%80%9A%E8%BF%87oauth%E7%99%BB%E5%BD%95)
命令后开启配置
```
openclaw configure
```
![选择model配置](/assets/img/posts/小龙虾从入门到入土/PixPin_2026-03-19_14-43-17.png)

> -
minimax接入可以选择OAuth 或手动安装； 如果用前者方式，注意要enable 相应minimax插件：
```
openclaw plugins enable minimax-portal-auth
openclaw gateway restart
```
此外，只有订阅minimax token plan的才能以OAuth方式接入（本穷鬼就先用实名认证送的15代金券试验了🥲）

根据教程，这里选择手动安装，根据指引一步步来，输入账号API密钥就可以。
